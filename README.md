# Yu-Gi-Oh Deck Tier List Manager

## 프로젝트 소개

유희왕 덱 정보를 저장하고 관리할 수 있는 덱 관리 툴입니다.

사용자가 덱 정보를 입력하고 각 평가 항목에 점수를 부여하면 총점을 계산하여 자동으로 티어를 산정합니다.

데이터는 로컬 JSON 파일과 Supabase 서버에 저장되며, 서버 장애 시에도 로컬 캐시를 통해 동작할 수 있도록 설계했습니다.

## 기술 선택 이유

### DDD 구조 적용

도메인 로직을 플랫폼 및 저장소에 의존하지 않도록 분리하기 위해 적용했습니다.
이를 통해 Domain 계층은 로컬 저장소, Supabase, UI 변경 여부와 관계없이 재사용 가능하도록 설계했습니다.

### 클래스 분리
JSON 직렬화/역직렬화 혹은 점수 검증책임 이나 티어 계산 규칙들을
별도 클래스로 분리하여 단일 책임 원칙을 적용하여 유지보수성과 테스트 편의성을 높였습니다.

### Supabase 사용

HTTP Module을 활용한 REST API 통신 경험과 JSON 직렬화/역직렬화 경험을 얻기 위해 사용했습니다.

## 문제및 해결

### 1. 이미지 절대 경로 문제

#### 문제

데이터에 이미지 경로 저장시 절대 경로를 저장하여 패키징 후 다른 PC에서 이미지를 불러올 수 없었습니다.

#### 해결

이미지를 Saved 폴더로 복사한 후 파일명만 저장하도록 변경했습니다. 

실제 경로는 런타임에 조합하여 사용하도록 수정했습니다.

---

### 2. 서버 동기화 실패 문제

#### 문제

Supabase Policy 설정이 없어 서버 저장 및 조회가 실패했습니다.

#### 해결

서버 데이터를 로컬에 캐싱하는 구조를 도입했습니다.

* 서버 조회 → 로컬 캐싱
* 저장 → 로컬 저장 후 서버 전송

이를 통해 서버 장애 발생 시에도 로컬 데이터 기반으로 동작할 수 있도록 개선했습니다.

이후 Supabase Policy 설정을 추가하여 서버 문제도 해결했습니다.

## 프로젝트 회고

초기에는 Manager 클래스에 많은 책임을 집중시키는 구조로 개발했지만 유지보수 과정에서 문제 원인 파악이 어려웠습니다.

이후 Serializer, Rule, Repository, Domain 클래스로 책임을 분리하면서 코드 가독성과 유지보수성이 향상되었고 객체 설계의 중요성을 경험할 수 있었습니다.
